У меня в Telegram живёт система из шести AI-агентов. Не один бот — а целая команда, где каждый занимается своим делом.

## Архитектура

Главный агент — стратег и координатор. Он принимает мои сообщения, разбивает задачи на этапы и раздаёт их пяти специализированным агентам:

**Менеджер** сидит в клиентском боте. Принимает сообщения, пересылает мне, сохраняет историю в CRM. Клиент пишет — я вижу сообщение через секунду с полным контекстом: имя, предыдущие обращения, статус.

**Безопасник** раз в неделю проверяет сервер. SSH-ключи, открытые порты, обновления, подозрительная активность. Присылает отчёт с конкретными рекомендациями.

**Архитектор** делает code review. Когда я или другой агент пишут код, архитектор проверяет структуру, находит потенциальные проблемы, предлагает рефакторинг.

**Оптимизатор** каждое утро ищет что автоматизировать. Анализирует повторяющиеся действия и предлагает скрипты или процедуры.

**Улучшатель** правит тексты. Это не один агент, а пайплайн из четырёх параллельных критиков.

## Четыре критика за текстом

Когда я прошу написать статью, запускается пятиэтапный процесс:

1. **Извлечение фактов** из рабочих заметок
2. **Исследование** — поиск аналогов, цен конкурентов, проверка актуальности
3. **Черновик** — первая версия текста
4. **Четыре критика параллельно:**
   - Детектор клише — ищет шаблонные фразы и AI-штампы
   - Анализатор ритма — проверяет разнообразие длины предложений
   - Проверка конкретики — требует числа, даты, имена вместо абстракций
   - Фактчекер — проверяет утверждения через поиск
5. **Финализация** — интеграция всех замечаний в итоговый текст

Критики работают независимо друг от друга. Каждый видит только черновик, не знает о замечаниях коллег. Это даёт честную, непредвзятую обратную связь.

## Как это выглядит на практике

Три примера из реальной работы.

**CRM-бот для автосервиса.** Написал в чат: «Сделай CRM-бота для автосервиса DSG». Главный агент разбил задачу на пять этапов: база данных, CRUD-операции, мультитенантность, управление заказами, тестирование. Каждый этап — отдельный субагент на дешёвой модели. Через несколько часов — рабочий продукт: SQLite, роли, аналитика, VIN-поиск, wizard создания заказов.

**Выставление счёта.** Написал: «Выстави счёт клиенту Оливия». Агент сгенерировал PDF договора, загрузил в систему электронного документооборота, создал счёт через API банка, отправил клиенту через бота. Ни одного клика в браузере.

**Статья для блога.** Написал: «Напиши статью про пайплайн критиков». Пять фаз отработали за 20 минут. Четыре критика нашли 11 замечаний: пассивные конструкции, повторяющийся ритм, отсутствие конкретных метрик, непроверенные утверждения. Все исправлены в финальной версии.

## Экономика

Субагенты работают на дешёвых моделях. Главный координатор — на мощной. Стоимость пайплайна одной статьи с четырьмя критиками — меньше доллара. CRM-бот от идеи до рабочего продукта — несколько долларов за API.

Вся система живёт в одном окне Telegram. Никаких дашбордов, дополнительных вкладок, переключений между приложениями. Пишу сообщение — получаю результат.

Бот работает 24/7. Я — нет. И в этом весь смысл.
