## Проблема: контекст теряется

У маркетингового агентства десятки клиентов. У каждого — свой проект, свои реквизиты, своя история переписки.

Когда ставишь задачу помощнику — будь то человек или AI — он спрашивает: «Какой клиент? Какой ИНН? А что там было в последнем сообщении?»

Это трата времени. Каждый раз.

## Что мы построили

CRM-систему, где задача создаётся **в контексте клиента**. Не просто «отправь счёт», а:

```
Клиент: Анна (ИП Иванова А.В.)
ИНН: 7712345678
Статус: клиент
Проект: Продвижение сайта — В работе, 50 000 ₽

Последние сообщения:
  Клиент: Пришлите счёт, пожалуйста
  Бот: Счёт №7 от 05.02.2026.pdf

Задание: Отправь счёт в ЭДО
```

AI-агент получает это и **сразу действует**. Не задаёт вопросов. Не ищет реквизиты. Не перечитывает 50 сообщений.

## Как это работает

### Канбан-доска

У нас 7 колонок: Лид → Контакт → В работе → Ожидание → Пауза → Готово → Отмена.

Каждая карточка — это клиент + задача + дедлайн + следующий шаг.

Свайп между колонками. Долгое нажатие — быстрая смена статуса. Всё на телефоне, в PWA-приложении.

### Создание задачи

Нажимаешь «+ Добавить задачу» в любой колонке:

1. **Выбираешь клиента** — поиск с автодополнением из базы
2. **Пишешь задачу** — «Отправить акт за январь»
3. **Описание** — детали, если нужны
4. **Дедлайн** — когда должно быть готово

Задача привязана к клиенту. При передаче AI-агенту — сервер **автоматически** подтягивает:

- Имя, компанию, ИНН клиента
- Текущий проект и его статус
- Последние 5 сообщений из чата

### AI в чате

Прямо в переписке с клиентом можно дать задание AI. Пишешь `@ai отправь в ЭДО счёт` — и агент:

1. Видит, с кем чат (имя, реквизиты)
2. Видит, какой файл был отправлен
3. Загружает счёт в систему ЭДО
4. Отчитывается прямо в чате

Без переключения между вкладками. Без копирования ИНН. Без «а какой именно счёт?»

## Чат с клиентом

Чат в CRM — не просто мессенджер. Он умеет:

- **Ctrl+V** — вставить скриншот из буфера, добавить подпись, отправить одним сообщением
- **Ссылки** — автоматически кликабельные
- **Файлы** — PDF, фото, видео через кнопку-скрепку
- **Реквизиты** — карточка клиента с ИНН, счётом, адресом
- **Светлая/тёмная тема** — переключатель, сохраняет выбор

## Экономия токенов

Раньше: AI получал голую задачу → делал 2-3 запроса к API → читал переписку → тратил ~500 лишних токенов.

Сейчас: контекст собирается на сервере. AI получает всё в одном пакете. **Ноль дополнительных запросов.**

На 10 задач в день это экономит ~5000 токенов. За месяц — ощутимо.

## Технические решения

- **PWA** — работает с домашнего экрана как нативное приложение
- **Две темы** — CSS-переменные, переключение без перезагрузки
- **Оффлайн** — сообщения кешируются, отправляются при восстановлении связи
- **Push-уведомления** — мгновенное оповещение о новых сообщениях клиентов
- **ЭДО интеграция** — отправка документов через СБИС прямо из чата

## Итого

CRM для маркетингового агентства, которая:

1. Хранит всё о клиенте в одном месте
2. Позволяет ставить задачи с контекстом
3. AI-агент работает без уточняющих вопросов
4. Экономит время и токены

Всё это — на своём сервере, под полным контролем. Никаких подписок на SaaS.
