Утро. Открываю Claude Code в терминале — и первые пятнадцать минут трачу на объяснения. Какой проект, что сделали на прошлой неделе, почему перешли на S3 для картинок. Пятнадцать минут — а я ещё ничего не сделал.

## Проблема

У AI нет памяти между сессиями. Закрыл терминал — всё исчезло. Решения, договорённости, прогресс. При трёх сессиях в день я терял около часа только на то, чтобы вернуть ассистента в контекст. Час чистых потерь — каждый день.

Вторая боль — задачи растворяются. «Давай доделаем завтра» — а завтра ни я, ни Claude не помним что именно. Нет места, где видно что в работе и что ждёт.

GitHub Issues подошёл потому, что это и таск-трекер, и внешняя память в одном месте. Описание задачи, комментарии с прогрессом, история решений — всё в одном issue. Claude Code работает с этим через gh CLI прямо из терминала.

## Утро — доска задач

Моё рабочее утро начинается с доски «Клиенты» в GitHub Projects. Колонки стандартные — Todo, In Progress, Done. Сейчас в работе четыре задачи: правки каталога на клиентском сайте, SEO-тексты для автосервиса, новая страница услуг для клиники и баг с PDF-отчётами.

Каждая задача — issue с описанием и комментариями. Claude видит эту доску. Когда я начинаю сессию, он читает issues и сразу понимает где мы остановились.

Конкретный пример — issue #47: «Каталог не обновляется после добавления товара в Strapi». В комментариях три записи. Первая: пробовали очистить кэш Next.js revalidate — не помогло. Вторая: нашли что Strapi 5 создаёт две записи — черновик и публикацию, фильтр тянул черновики. Третья: осталось поправить фильтры по категориям. Claude читает комментарии и продолжает именно с фильтров.

## День — работа

Я пишу в терминале: «возьми следующую задачу из In Progress». Claude через gh CLI смотрит доску, находит issue, подгружает описание и комментарии. Работает. Пишет прогресс прямо в issue — что сделал, какие решения принял.

Появилась новая задача? Создаёт issue и ставит на доску в Todo. Ничего не висит в голове — всё зафиксировано.

В пятницу клиент написал: «Каталог ок, теперь хочу русифицировать админку Strapi». Я создал issue, описал требования и закрыл ноутбук. В понедельник открыл новую сессию — Claude прочитал issue и начал с локализации. Не с вопроса «а что за проект?».

## Вечер — результат

К концу вторника закрыты: синхронизация изображений с S3, правка шапки сайта автосервиса, генерация PDF-аудита. У каждой issue — финальный комментарий: что сделано, какой коммит, как проверить на проде. Доска обновлена.

Завтра — новая сессия, может другой AI-инструмент. Контекст лежит в issues, не в памяти конкретного ассистента.

GitHub в январе 2026 тоже взялся за проблему потери контекста — встроил в Copilot систему памяти между сессиями. Подход другой, но задача та же: чтобы AI не начинал каждый раз с чистого листа.

## Зачем это маркетологу

GitHub выглядит как что-то для программистов. По факту — просто список задач, где каждая помнит свою историю. Notion так не умеет — там AI не заглянет в заметки сам.
