## 1. Налоговые режимы и лимиты (2026)

### 1.1 Самозанятость (НПД)
**Источник**: [Контур](https://kontur.ru/articles/4818), [Яндекс.Касса](https://pay.yandex.ru/blog/articles/samozanyatye-izmeneniya-2026), [Консоль](https://konsol.pro/blog/prava-samozanyatyh)

| Параметр | Значение |
|----------|----------|
| **Налоговые ставки** | 4% — с физлиц; 6% — с юрлиц и ИП |
| **Налоговый бонус** | −1% ставки в первый год (3% и 5% соответственно) |
| **Годовой лимит доходов** | **2,4 млн ₽** (обнуляется 1 января каждого года) |
| **Платёж налога** | До 28-го числа следующего месяца |
| **Сроки действия** | Эксперимент продлён до 31.12.2028 |
| **Социальные взносы** | Добровольные: 1344–1920 ₽/мес (за сумму 35–50 тыс ₽) |

**Ограничение**: при превышении 2,4 млн ₽ в год НПД автоматически отменяется с начала месяца превышения.

---

### 1.2 ИП на УСН 6% («Доходы»)
**Источник**: [Моё Дело](https://www.moedelo.org/club/nalogovyj-uchet/usn-dohody-dohody-minus-rashody-2026), [Главбух](https://www.glavbukh.ru), [СБИС](https://saby.ru)

| Параметр | Значение |
|----------|----------|
| **Лимит дохода** | **490,5 млн ₽** в год |
| **Лимит стоимости ОС** | 218 млн ₽ |
| **Численность сотрудников** | До 130 человек |
| **Налоговая ставка** | 6% (от дохода) |
| **Фиксированные взносы за себя** | **57 390 ₽** (срок: 28.12) |
| **Доп. взнос** | 1% с дохода свыше 300 000 ₽ |
| **Верхний предел доп. взноса** | **321 818 ₽** |
| **Уменьшение налога на взносы** | Без сотрудников: до нуля; с сотрудниками: до 50% |

**НДС: КРИТИЧЕСКОЕ ИЗМЕНЕНИЕ 2026** ([Моё Дело](https://www.moedelo.org/club/nalogovyj-uchet/perehod-s-usn-na-nds-v-2026-godu))
- Порог освобождения от НДС снижен с **60 млн до 20 млн ₽** (доход за 2025 год)
- При превышении 20 млн ₽ в текущем году: обязанность по НДС с 1-го числа следующего месяца
- Ставки: 5% и 7% для доходов свыше 20 млн ₽ (на период 20–250 млн)
- Поэтапное снижение: 15 млн ₽ (2027), 10 млн ₽ (2028 и далее)
- Законодатель: [ФЗ-425 от 28.11.2025](https://www.consultant.ru)

---

### 1.3 ИП на АУСН (автоматизированная УСН)
**Источник**: [Контур](https://kontur.ru/articles/4535), [Главбух](https://www.glavbukh.ru/art/391259-ausn-s-2026-goda-izmenyat-li-limit)

| Параметр | Значение |
|----------|----------|
| **Лимит дохода** | **60 млн ₽** (до 31.12.2027) |
| **Численность сотрудников** | До 5 человек |
| **Стоимость ОС** | До 150 млн ₽ |
| **Ставки** | 8% (вместо 6% на УСН); 20% (вместо 15% на УСН/расходы) |
| **Страховые взносы** | **Не платятся** за себя и сотрудников |
| **Взнос на травматизм** | 2 959 ₽/год за сотрудников |
| **Преимущество** | Упрощённая отчётность, нет НДС |

**Вывод для агентства**: АУСН привлекателен отсутствием взносов, но выше налоговая ставка.

## 2. API и интеграция с банками

### 2.1 Точка Банк API (ПРИОРИТЕТ ДЛЯ ВЛАДЕЛЬЦА)
**Документация**: [developers.tochka.com](https://developers.tochka.com/docs/tochka-api/)  
**Источник**: [Точка. API](https://allo.tochka.com/vznosy-ip)

**Основные эндпоинты**:

| Операция | Описание | Метод |
|----------|---------|-------|
| **Выписка** | Получить выписку по счёту за период (остаток начала/конца, операции) | `Init Statement` + `Get Statement` |
| **Балансы** | Получить текущий баланс счёта | `GET /open-banking/v{version}/accounts/{accountId}/balances` |
| **Платежи** | Создать черновик платежа, подписать, отправить | `POST` (draft → sign) |
| **Счета** | Создавать счета на оплату (PDF), отправлять контрагентам | `POST /invoicing/` |
| **Закрывающие документы** | Создавать акты, счета-фактуры в ЭДО | Интеграция через Диадок API |
| **СБП** | Регистрация, QR-коды, возвраты платежей через Систему быстрых платежей | `POST /sbp/` |
| **Вебхуки** | Уведомления о входящих/исходящих платежах | Webhooks |

**Возможности**:
- Неограниченное подписание платежей ЭЦП из интегрированного сервиса
- Получение списка счетов и балансов
- Создание и отправка счетов-фактур

**SDK**: доступны готовые библиотеки на PHP, Python, TypeScript ([GitHub](https://github.com/FatherOctber/tochka-sdk), [GitHub](https://github.com/WhiteApfel/tochka_api))

### 2.2 Тинькофф Бизнес API (T-API)
**Документация**: [developer.tbank.ru](https://developer.tbank.ru/)  
**Источник**: [Тинькофф](https://www.tbank.ru/business/open-api/), [Записки разработчика](https://byurrer.ru/tinkoff-business-api)

| Операция | Описание |
|----------|---------|
| **Выписка** | Метод получения операций по счёту за период (данные с июня 2023) |
| **Платежи** | Создание черновиков, платежных поручений, платежи через СБП |
| **Счета** | Выставление счетов на оплату в PDF |
| **Безопасность** | Mutual TLS для платежей; лимиты ЭЦП на все платежи (веб, мобильное, API) |

**Преимущества**:
- Бесплатная интеграция (в отличие от других банков)
- Грамотная документация, легко работать
- Контакт: openapi@tbank.ru

### 2.3 Контур.Эльба API
**Источник**: [Контур](https://kontur.ru), [Solo IT](https://solo-it.ru/blog/integratsiya-kontur-elba-api-elby-dlya-svyazi-s-amocrm-i-bitriks24-chast-1.html)

**Документация**: [moedelo.org/api](https://www.moedelo.org/landingpage/api/developers)  
**Источник**: [API Portal](https://apiportal.ru/catalog/mojo-delo-api/)

### 2.6 Диадок (ЭДО) API
**Документация**: [developer.kontur.ru/doc/diadoc-api](https://developer.kontur.ru/doc/diadoc-api/)  
**Источник**: [Диадок](https://www.diadoc.ru), [Контур](https://kontur.ru/diadoc)

**Основные возможности**:
- Обмен юридически значимыми электронными документами между организациями
- HTTP-based API, интеграция с любыми системами (1С, SAP, Oracle, MS Dynamics и др.)
- Готовые интеграции: 1С, SAP, Docsvision, Directum, Парус, БЭСТ-5, EnDocs

**Поддерживаемые документы**:
- Счета-фактуры, УПД (универсальные передаточные документы)
- Акты, накладные
- Транспортные ЭДО (с 01.09.2026 обязательны в грузоперевозках)

**Версионирование**: документы версии 5.03 для МЧД (машиночитаемые доверенности)

## 3. Что AI-агент может делать легально

### 3.1 Операции БЕЗ подписи (полная автоматизация)

| Операция | Описание | Ограничения |
|----------|---------|-----------|
| **Распознавание документов** | OCR сканов/фото счетов, актов, чеков — извлечение данных | Погрешность OCR, требует проверки |
| **Обработка выписок** | Загрузка пакетов PDF, извлечение реквизитов, отправка в учёт | Автоматическое распределение по категориям |
| **Сверка по выписке** | Сопоставление поступлений с договорами, поиск расхождений | Поиск несовпадений, уведомления |
| **Формирование книги доходов** | Автоматический расчёт доходов за период (НПД) | Для режима НПД; включает валиды |
| **Напоминания о налогах** | Расчёт сумм, сроков уплаты взносов (ИП) и налогов (НПД) | На основе режима и выручки |
| **Создание черновиков счетов** | Генерация счётов на оплату, НПД-чеков (в приложении ФНС) | Без подписи — готовы для печати/отправки |
| **Формирование актов выполненных работ** | Шаблонизированные акты на основе выполненных услуг | Требует подтверждения человеком перед подписью |

### 3.2 Операции, ТРЕБУЮЩИЕ подписи человека (ЭЦП)

| Операция | Почему | Документ |
|----------|--------|----------|
| **Подписание счетов-фактур** | Налоговый документ → квалифицированная подпись (КЭП) | ТДП (счёт-фактура) |
| **Подписание актов** | Договор на услуги → КЭП обеих сторон | Акт выполненных работ |
| **Подписание договоров** | Юридически значимый документ → КЭП | Договор, счёт-договор |
| **Сдача деклараций в ФНС** | Налоговая отчётность → КЭП главного бухгалтера или ИП | 3-НДФЛ, УСН-2, ЕСХН и др. |
| **ЭДО через Диадок** | Обмен документами с контрагентами → подписант (сотрудник с МЧД) | Счёта-фактуры, УПД в Диадоке |
| **Платежи через банк** | Авторизация платежей → ЭЦП (встроены в Точка API, Т-API) | Payment order, платёжное поручение |

**Исключение НПД**: плательщик НПД **не требует** ЭЦП для выставления налоговых чеков в мобильном приложении.

---

### 3.3 Стратегия для агентства

**Режим «максимум автоматизации»** (ИП на УСН 6% или НПД):
1. **AI генерирует**: счета, акты (черновики), платёжные поручения, напоминания о взносах
2. **Собственник подписывает**: декларации УСН, счета-фактуры (если есть НДС), договоры
3. **API интеграция**: Точка/Тинькофф → выписка → сверка → уведомления

## 4. Типовые договоры для маркетинговых услуг

**Источник**: [Моё Дело](https://www.moedelo.org/blanki/item/dogovor-ob-okazanii-marketingovyh-uslug), [КонсультантПлюс](https://www.consultant.ru), [DataDoc](https://datadoc.ru)

### 4.1 Структура договора (ГК РФ, гл. 39 — «Возмездное оказание услуг»)

| Документ | Цель | Когда использовать | Подпись обязательна |
|----------|------|-------------------|------------------|
| **Счет** | Предложение об оплате (до заключения договора) | Одноразовые, простые услуги | Нет (счёт в одну сторону) |
| **Оферта** | Публичное предложение условий (акцепт = оплата) | Массовые услуги (как на сайте) | Не требует подписи для акцепта |
| **Счет-оферта** | Счет + предложение условий договора | Пробные услуги, небольшие объёмы | При акцепте (оплата) договор возникает |
| **Счет-договор** | Счет + подписанный договор | Стандартные маркетинговые проекты | **ДА — обе стороны** (ИП и заказчик) |
| **Счет-фактура** | Налоговый документ (при НДС) | При доходе свыше 20 млн ₽ (2026) на УСН | **ДА — продавец** (ЭЦП) |

### 4.2 Закрывающие документы

| Документ | Назначение | Подпись |
|----------|-----------|--------|
| **Акт выполненных работ** | Подтверждение оказания услуг, отсутствие претензий | Обе стороны (КЭП на ИП и представитель заказчика) |
| **Накладная** | Передача материальных активов (редко в маркетинге) | Обе стороны |
| **Приложение к счету-договору** | Детали выполненных работ за период | Обе стороны |

### 4.3 Рекомендуемый поток для агентства

```
1. Звонок / Email   → Предварительное обсуждение
   ↓
2. Счет-оферта     → Отправка юрлицу; если оплата → договор считается заключённым
   или
   Счет-договор    → Подписание обеими сторонами + оплата (более формальный путь)
   ↓
3. Оказание услуг  → Выполнение работ (контент, реклама, аналитика)
   ↓
4. Акт выполненных → Подписание на основе выполненной работы
   работ (с ЭЦП)   → Фиксирует завершение этапа
   ↓
5. Счет-фактура   → Если доход свыше 20 млн ₽ за год (НДС)
   (если НДС)       → Подписание электронной подписью
   ↓
6. Архив           → Сохранение для налоговой
```

## 6. Рекомендации по реализации для AI-COO

### 6.1 Приоритет 1: Точка Банк (у владельца счёт)
- **Интегрировать**: выписка (ежедневно) → сверка по договорам → уведомления о задержках платежей
- **API**: `GET /open-banking/v{version}/accounts/{accountId}/balances` + `Init/Get Statement`
- **Цель**: автоматическое отслеживание поступлений от клиентов

### 6.2 Приоритет 2: Документооборот (счета, акты)
- **Шаблоны**: счета-черновики и акты → хранение в локальной БД
- **Подготовка**: AI заполняет данные, генерирует PDF
- **Подпись**: собственник просматривает, подписывает ЭЦП через Точка API или локальный инструмент

### 6.3 Приоритет 3: Налоговая отчётность
- **НПД** (если используется): автоматическая книга доходов, напоминания о сроках чеков
- **УСН**: напоминания о взносах (57 390 ₽ + 1% сверх 300k), сроках уплаты (28 декабря)
- **НДС-мониторинг**: при превышении 20 млн ₽ — уведомление о необходимости подачи деклараций

### 6.4 Приоритет 4: ЭДО (дополнительный функционал)
- **Диадок API** для обмена счетами-фактурами с крупными клиентами (юрлицами)
- **МЧД** (машиночитаемая доверенность) для подписей сотрудников от имени ИП
