# Регламент первой линии клиентской коммуникации

Кто работает по этому регламенту: первая линия. **До найма человека первую линию исполняет AI-COO** (omp-сессия проекта eldar.marketing); при найме помощника документ становится его онбордингом. Цель: клиент всегда получает быстрый ответ, ни одна задача не теряется, Эльдар не участвует в рутинной переписке.

## Главное правило

**Задачи не существует, если её нет на борде.** Любая просьба клиента — из чата, звонка, письма, «на бегу» — в течение часа превращается в issue на борде «Клиенты». Устное обещание без issue = нарушение регламента.

## SLA

| Событие | Норматив |
|---|---|
| Ответ клиенту «принято» | ≤ 4 рабочих часов |
| Рутинная задача (в скоупе пакета) | ≤ 48 ч до Done |
| Задача вне скоупа | ≤ 24 ч на ответ «оценим, вернёмся со сметой» |
| Ответ на отзыв на Картах | ≤ 48 ч |
| Инцидент «сайт лежит» | немедленно: эскалация Эльдару + рестарт по деплой-схеме проекта (скилл `go` → deploy) |

Рабочие часы: пн–пт 10:00–19:00 МСК. Вне окна отвечаем в начале следующего рабочего дня.

## Поток задачи

1. **Принять**: ответить клиенту шаблоном «Принято» (№1).
2. **Занести**: issue в репо клиента (routing — таблица в конфиге агентства), добавить на борд → **Inbox**, приоритет: P0 — сломано/деньги, P1 — обычная, P2 — «когда-нибудь».
3. **Классифицировать**: в скоупе пакета → Todo. Вне скоупа → шаблон №4 + метка на оценку Эльдару.
4. **Исполнить**: по SOP пакета сам / задача Claude Code по скиллу проекта. Готово → **Review**.
5. **После ревью**: отправить результат клиенту (шаблон №3) → **Done**. Ждём клиента/материалы → **Waiting** + напоминание клиенту раз в 3 рабочих дня.

## Шаблоны ответов

**№1 · Принято:**
> Принял(а), спасибо! Задачу завёл(а) — до конца дня вернусь с точным сроком. Если понадобятся материалы от вас — напишу отдельно.

Срок называем **после** классификации (шаг 3): в скоупе → «сделаем до {дата = +2 раб. дня}», вне скоупа → шаблон №4.

**№2 · Нужно уточнение:**
> Чтобы сделать сразу правильно, уточните, пожалуйста: {один конкретный вопрос}. Как ответите — беру в работу, срок {N} рабочих дня после ответа.

**№3 · Готово:**
> Готово: {что сделано, ссылка/скрин}. Посмотрите, пожалуйста — если что-то поправить, напишите, внесём в течение дня.

**№4 · Вне пакета:**
> Хорошая идея! Это выходит за рамки абонентского пакета — подготовим оценку по срокам и стоимости и вернёмся до {дата, ≤24 ч}. Само собой, без работ «в счёт» — только после вашего ок.

**№5 · Задержка (отправлять ДО срыва срока):**
> Небольшая задержка по задаче «{название}»: {причина в одну строку}. Новый срок — {дата}. Извините, держу в курсе.

## Тон

Коротко, по делу, без канцелярита и без «мы стараемся». Конкретика вместо обещаний: не «скоро сделаем», а «сделаем до среды». Признаём ошибки сразу и с новым сроком. На «поправь батончик» не ворчим — заносим и делаем.

## Что первая линия НЕ делает

- Не обещает новых работ, скидок и сроков вне SLA — это Эльдар.
- Не обсуждает цены и договоры — переадресация Эльдару.
- Не деплоит в прод и не мержит в main — только через Review.
- Не отправляет клиенту цифры/факты, не проверив источник (отчёт, Метрика).
- Не ведёт переписку с личного аккаунта — только с рабочего.

## Каналы и доступы

- Переписка с клиентами: рабочий Telegram и почта `i@eldarmarketing.ru`. Личные аккаунты — нельзя.
- Вся входящая коммуникация (чат, звонок, письмо) фиксируется issue на борде — источник правды один.
- Доступы первой линии: борд «Клиенты» (GitHub Project), почта, рабочий TG. Без доступа к: банку, договорам, проду (деплой — только через Review).

## Эскалация Эльдару (немедленно)

Сайт/CRM лежит · клиент недоволен и пишет об отказе · вопрос денег (счета, цены, долги) · юридика · любая просьба дать кому-то доступ.
